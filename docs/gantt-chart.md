# Project Gantt Chart

This Gantt chart summarises the main project activities from Week 1 to Week 11 for the Restaurant Ordering and Management System.

| Week | Project Activity | Main Output |
|---|---|---|
| Week 1 | Project initiation and topic selection | Project idea, problem statement, initial scope |
| Week 2 | Requirements analysis | Functional and non-functional requirements, actors, acceptance criteria |
| Week 3 | Literature review and fact-finding | Literature themes, project gap, recommendation justification |
| Week 4 | System analysis and design | Use case diagrams, ERD, architecture design, page flow |
| Week 5 | Requirements refinement | Final feature scope, customer/admin requirements, implementation plan |
| Week 6 | Technical design finalisation | Backend structure, API design, database schema planning, security design |
| Week 7 | Backend development | Express API, MongoDB models, authentication, initial API testing |
| Week 8 | Frontend development and integration | Customer pages, admin pages, cart, booking, reviews, API integration |
| Week 9 | Testing and evaluation progress | Postman testing, frontend testing, defect fixing |
| Week 10 | Product refinement and documentation | Online payment refinement, testing report, submission evidence |
| Week 11 | Final review and viva preparation | Final evaluation, reflection, GitHub/report cleanup, demo checklist |

## Critical Path

The critical path for the project was:

1. Requirements analysis
2. Literature review and design
3. Backend implementation
4. Frontend/API integration
5. Testing and refinement
6. Final evaluation and documentation

Any delay in backend development would have affected frontend integration because most customer and admin pages depend on API responses. Testing was therefore planned after integration but also started during backend development using Postman.
